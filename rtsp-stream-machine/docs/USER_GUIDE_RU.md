# RTSP Stream Machine: Пользовательская Инструкция

## 1. Что делает сервис

Сервис берет видеофайлы из `videos/` и публикует RTSP-потоки через MediaMTX.

Пример RTSP-ссылки:

```text
rtsp://<SERVER_IP>:8554/cam001
```

## 2. Где работать

Web UI:

```text
http://<SERVER_IP>:8090/
```

`/ui` и `/ui/` автоматически перенаправляют на `/`.

## 3. Библиотека видео

В блоке `Библиотека видео` можно:

- загрузить видео (`+ Загрузить`)
- отключить/включить видео
- удалить видео (через модалку подтверждения)

Отображение:
- блоки `Активные стримы` и `Библиотека видео` показаны плитками (галерея)
- сортировка по дате добавления: новые сверху
- на странице показывается по 6 карточек, остальные доступны через пагинацию

Фильтры:
- в `Активные стримы` есть кнопка `Типы стримов` (список по клику)
- в `Логи` есть кнопка `Типы логов` (фильтр по уровням `info/warn/err`)
- в `Логи` показывается по 30 записей на страницу, остальное через пагинацию

Статус `включено/отключено`:

- `включено` — файл доступен для создания/запуска стримов
- `отключено` — файл не используется в мастере запуска

## 4. Создание и запуск стримов

Кнопка в блоке `Активные стримы`:

- `Запустить стрим`

Откроется мастер с 3 сценариями:

1. `Single` — один стрим из одного видео
2. `Playlist` — один стрим из нескольких видео (по кругу)
3. `Multi` — отдельный стрим на каждый выбранный файл

Поля мастера:

- имя стрима (для `Single`/`Playlist`)
- префикс (для `Multi`, например `cam_auto_`)
- кодек (`auto`, `transcode`, `copy`)
- запуск сразу после создания

Список видео в мастере:
- показывается вертикальным списком с нумерацией
- есть поле `Поиск видео` для быстрого фильтра по имени файла
- `Выбрать все` работает для текущего (видимого) отфильтрованного списка

`auto` (по умолчанию):
- backend проверяет файл через `ffprobe`
- если файл RTSP-совместим -> используется `copy`
- иначе -> `transcode`

## 5. Управление активными стримами

Для каждого активного стрима:

- `Остановить`
- `Перезапустить`

Остановленные стримы в UI не показываются.
Если стрим остановлен более 7 дней, он автоудаляется из конфигурации.

## 6. Расписания

Кнопка:

- `+ Быстрый таймер`

Можно создать one-time правило запуска/остановки/удаления для выбранных стримов по дате/времени.

## 7. Логи и состояние

- `Состояние системы` — API, MediaMTX, Планировщик
- `Логи` — ключевые ошибки/предупреждения
- `Диск` — информация по разделам и предупреждение о низком свободном месте
- При включенном watchdog (`AUTO_RECOVER_STREAMS=true`) упавшие активные стримы поднимаются автоматически.

## 8. Проверка потока

Через ffprobe:

```bash
ffprobe rtsp://<SERVER_IP>:8554/<stream_name>
```

Через VLC:

```text
rtsp://<SERVER_IP>:8554/<stream_name>
```

## 9. Частые проблемы

### `404 create-from-videos`

Запущен старый backend. Нужен перезапуск `app/api_server.py`.

### VLC не открывает поток

Проверить:

- запущен ли стрим в UI
- порт `8554/tcp` открыт
- нет ли ошибок в `logs/<stream_name>.log`
